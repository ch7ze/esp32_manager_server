# Projekt-spezifische Regeln für Master Seminar

## Dateierstellung - ABSOLUT VERBOTEN
- **NIEMALS** neue Dateien erstellen ohne explizite Erlaubnis
- **NIEMALS** neue Python-Skripte (.py) erstellen
- **NIEMALS** Analyse-Skripte, Vergleichs-Skripte, Helper-Skripte erstellen
- **NIEMALS** Test-Scripts, Demo-Scripts oder Beispiel-Dateien erstellen
- **NIEMALS** Dokumentationsdateien (*.md, README, etc.) erstellen
- **NIEMALS** Tutorial- oder Erklärungs-Scripts erstellen
- **NUR** existierende Dateien lesen und modifizieren

## Workflow bei Bedarf für neue Dateien
1. **STOPPEN** - Nicht einfach Datei erstellen
2. **VORSCHLAGEN** - Im Chat erklären, welche Datei hilfreich wäre
3. **WARTEN** - Auf explizite Zustimmung des Users warten
4. **DANN** erst erstellen

Beispiel:
❌ FALSCH: Einfach `compare_results.py` erstellen
✓ RICHTIG: "Ich könnte ein Vergleichs-Skript `compare_results.py` erstellen. Soll ich das machen?"

## Erlaubte Operationen
- Existierende Dateien LESEN (Read tool)
- Existierende Dateien MODIFIZIEREN (Edit tool)
- Bestehende Code-Dateien verbessern
- Bugs in existierendem Code fixen

## Verbotene Operationen (ohne explizite Erlaubnis)
- **ALLE** neuen Dateien jeglicher Art
- Neue .py Dateien (inkl. Analyse-, Vergleichs-, Helper-Skripte)
- **Python-Code in Bash-Befehlen (`python -c "..."`) - NIEMALS, keine Ausnahmen**
- **Komplexe Analysen über Bash - stattdessen Dateien direkt lesen mit Read tool**
- Neue .md Dateien für Dokumentation oder Zusammenfassungen
- Neue .txt Dateien für Erklärungen oder Berichte
- Neue .json Dateien für Konfiguration oder Daten
- Jegliche "example_*.py", "test_*.py", "demo_*.py", "compare_*.py", "analyze_*.py" Dateien
- README-Dateien oder ähnliche Dokumentation

## Kommunikation
- **ALLE** Erklärungen, Analysen und Beispiele NUR im Chat
- **ALLE** Vergleiche und Zusammenfassungen direkt im Chat ausgeben
- Code-Änderungen direkt in bestehenden Dateien
- Wenn Demonstration nötig: Code-Beispiele im Chat zeigen, nicht als Datei
- Bei komplexen Analysen: Ergebnisse formatiert im Chat präsentieren
- Nur eine Lösung vorschlagen, wenn die Ursache des Problems im code verifiziert wurde

## Ausnahmen (NUR mit expliziter User-Erlaubnis)
- User sagt explizit: "erstelle eine neue Datei..."
- User sagt explizit: "schreibe ein Script für..."
- User sagt explizit: "mache eine Dokumentation..."
- User sagt explizit: "ja, erstelle das"
